# 02. MVP 기능 정의 & 로드맵

## MVP 판단 기준
1. 서비스 핵심 가치(고유 운영 규칙 시스템화)와 직접 연결
2. 없으면 서비스가 성립하지 않음
3. 1인 개발로 완성 가능한 범위

## Phase 1 (MVP) — 반드시 구현
| # | 기능 | 비고 |
|---|---|---|
| ① | 회원가입/로그인 (가입코드=즉시가입, 학번대조) | 기반 |
| ② | 임원/부원 권한 구분 (ROLE_ADMIN/MEMBER) | 기반 |
| ③ | 조 편성 + 조장 지정 + 정규활동 일정 설정 | 기반 |
| ④ | 가능 요일 수집 + 자동 조 배정 (preview/confirm) | ★ 엔진1 |
| ⑤ | 매일 활동 투표 자동 생성 + 참여 유형 분기 | ★ 엔진2 |
| ⑥ | 이월 처리 (보장 자격 이동) | ★ 엔진2 |
| ⑦ | 출석 + 월 4회 충족 자동 계산 (임원) | 엔진2 |
| ⑧ | 부원 본인 화면 (소속조 확인 + 투표) | 부원 |

## Phase 2 — 서비스 확장
공지 / 회비 납부 상태 관리 / 자유활동 기간 설정 UI / 푸시 알림

## Phase 3 — 고도화
이동 동행 관리 / 게시판 / 출석·투표 통계 / 조장 권한 분리 / 지난학기 조편성 복제 / 일반 투표(MT 등)

## 제외 (전 Phase 공통)
체육관비 금액 계산·정산 / PG 결제 / 구글폼 API / 카톡 자동초대 / 복식 매칭·랭킹

## MVP가 증명하는 것
- 가능 요일 기반 균형 자동 배정 (제약 만족 알고리즘)
- 매일 투표 생성·집계 자동화 (스케줄러)
- 조장 머릿속 이월 판단의 데이터화 (상태 전이 로직)
- 월 4회 충족 계산 자동화
