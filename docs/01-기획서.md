# 01. 프로젝트 기획서

## 1. 프로젝트 개요

- 서비스명 : SMASH
- 서비스 유형 : 배드민턴 동아리 활동 운영 관리 앱
- 개발 형태 : 개인 프로젝트 (기획·설계·개발 전담)
- 플랫폼 : iOS / Android (Flutter 크로스플랫폼)
- Frontend : Flutter / Riverpod / Dio
- Backend : Java 21 / Spring Boot 3.5 / Gradle
- DataBase : MySQL
- 인증 : JWT (Access / Refresh Token)
- API : RESTful API

### 개발 배경
대학교 배드민턴 동아리 부회장으로 운영을 담당하며 세 가지 구조적 문제를 경험했다.
1. 조 배정·출석 이월·타조 참여·매일 활동 투표 등 고유 운영 규칙을 처리할 전용 도구 부재 → 카카오톡·엑셀 수작업
2. 한 학기 100~250명 신규 부원 모집·가능 요일 취합·조 배정의 대규모 수작업
3. 활동 당일 이동 동행·정산이 조장 수작업에 의존

공통 원인은 운영 규칙과 구성원 데이터를 통합 처리할 전용 체계가 없다는 점이다.

### 프로젝트 목적
배드민턴 동아리 고유 운영 규칙(조 배정, 출석 이월, 타조 참여, 일일 활동 투표)과 운영 데이터를
하나의 플랫폼에서 관리하도록 시스템화하여, 임원의 반복 수작업을 줄이고 데이터 연속성을 확보한다.

## 2. 서비스 소개
- 문제 배경 : 평일 10개 조 편성, 조별 월 4회 보장, 당월 이월 제도, 타조 참여, 매일 투표가 맞물린
  운영 규칙 + 학기 100~250명 신규 모집·조 배정. 전용 도구가 없어 카톡·엑셀·구글폼 수작업.
- 개선 방향 : 매일 투표 생성·출석 이월 계산·타조 집계 자동화 / 신규 부원 모집~배정~활동기록 중앙화
- 서비스 가치 : 운영자(반복 업무 감소), 부원(본인 조·활동 직접 확인), 조직(임원 교체에도 데이터 연속)

## 3. 타겟 사용자
### Primary — 임원 (회장/부회장/총무/조장)
Pain Point : 모집·연락처저장·카톡초대 수작업 / 가능요일 취합·균형 배정 / 매일 투표 생성·집계(월 최대 40개)
/ 이월·충족 수기 계산 오류 / 회비 다단계 취합 / 데이터 분산으로 인수인계 어려움
Needs : 신규부원 통합관리, 자동 조배정, 투표 자동생성·집계, 충족현황 자동산정, 부원정보 중앙관리

### Secondary — 부원
Pain Point : 본인 조·활동일 확인 어려움 / 이월·타조 방법 불명확해 조장 문의 / 이동 동행·정산 기록 없음
Needs : 가능요일 제출, 매일 투표 참여(정규·이월·타조·불참), 이월 시 날짜 지정, 본인 소속조 확인

## 4. 문제 정의
기존 운영 방식과 한계:
- 신규 모집 : 구글폼 → 면접 → 연락처 수동저장 → 카톡 초대 (수작업, 누락)
- 조 배정 : 가능요일 별도 취합 → 임원이 10개 조 균형 수동 배정
- 출석·이월 : 임원·조장 수기 계산 (오류 → 신뢰 문제)
- 회비 : 조장 취합 → 총무 엑셀 (행사비 + 체육관비 4주치)
- 데이터 분산 : 구글폼·개인기기·메모·엑셀·카톡에 흩어짐 → 인수인계 불가

핵심 문제 : 도구 자체가 아니라, 조 배정~투표~이월~타조~당일이동까지 이어지는 고유 운영 규칙을
통합 처리할 전용 체계가 없어 임원이 여러 도구로 수작업하는 구조가 근본 문제.

## 5. 서비스 목표
운영자 : 반복 수작업 자동화 / 모집·조배정 효율화 / 운영 투명성 / 데이터 연속성
부원 : 운영 규칙 접근성 향상 (소속조·투표·이월을 앱에서 직접) / 활동 참여 편의성
