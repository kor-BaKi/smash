# 04. 핵심 로직 (의사코드 + 테스트)

이 문서는 SMASH의 두 엔진을 코드 직전 수준으로 정리한 것. 단위 테스트 필수.

## 엔진 1 — 자동 조 배정

입력: 미배정 부원들 + 각자의 가능 요일조(복수)
목표: 가능 조 중 하나에 배정, 조별 인원 최대한 균등
제약: 선택지 적은 부원부터 먼저 배정 (결정적)

```
function autoAssign(members, groups):
    for m in members: m.optionCount = count(m.availableGroups)
    sorted = sort(members, by=[optionCount ASC, id ASC])   # 결정적
    groupCount = { g.id: 0 for g in groups }
    assignments, unassigned = [], []
    for m in sorted:
        if m.availableGroups is empty:
            unassigned.add(m, reason=NO_AVAILABILITY); continue
        target = minBy(m.availableGroups, key=[groupCount[g], g.id])  # 인원 적은 조, 동점 id순
        assignments.add(m -> target); groupCount[target] += 1
    return { assignments, unassigned, groupCount }
```

설계 근거: 그리디 + 결정적(동점 id순) + O(N log N). N=250,G=10이라 충분.
"선택지 적은 부원 먼저"는 임원 수작업을 그대로 코드화한 것.

테스트:
- TC1 전원 전체조 가능 → 각 조 ±1
- TC2 1개만 고른 부원 → 해당 조 배정
- TC3 미제출 → unassigned(NO_AVAILABILITY)
- TC4 동일입력 2회 → 결과 동일(결정성)
- TC5 253명/10조 → 최대-최소 ≤ 2

## 엔진 2 — 이월/출석

### 공통 헬퍼
```
hasCarryoverOut(member, activity):   # 이 날 자격이 빠졌나?
    return exists Participation where user=member, type=CARRYOVER,
           carryover_target_activity_id = activity.id

hasRegularAttend(member, activity):  # 이 날 정규 참여했나?
    return exists Participation where user=member, activity=activity, type=REGULAR
```

### ① 버튼 분기 (D-1)
```
resolveButtons(member, activity):
    if activity.type == FREE: return [FREE_ATTEND, ABSENT]
    isMy = (member.group_id == activity.group_id)
    if isMy:
        if hasCarryoverOut(member, activity): return [CARRYOVER, OTHER_GROUP]  # 자격 빠진 날
        else: return [ATTEND, ABSENT]
    else: return [CARRYOVER, OTHER_GROUP]
```

### ② 이월 후보 (D-3)
```
getCarryoverCandidates(member, currentActivity):
    monthStart, monthEnd = monthRange(currentActivity.date)
    myActs = Activity where group=member.group_id, date in [monthStart,monthEnd], not cancelled
    candidates = []
    for act in myActs:
        if hasCarryoverOut(member, act): continue          # 이미 자격 뺀 날 제외
        if act.date > today: candidates.add(act, FUTURE)
        elif act.date < today:
            if not hasRegularAttend(member, act): candidates.add(act, PAST_ABSENT)
    return candidates   # 빈 배열이면 "이월 불가"
```

### ③ 충족 계산 (E-1, 임원 전용)
```
calculateFulfillment(member, year, month):
    range = monthRange(year, month)
    guaranteed = count Activity where group=member.group_id, date in range,
                 not cancelled, type=REGULAR
    fulfilled = count Participation where user=member,
                activity.date in range, type in [REGULAR, CARRYOVER]
    shortfall = max(0, guaranteed - fulfilled)
    return { guaranteed, fulfilled, shortfall }
# OTHER_GROUP, FREE_ATTEND, ABSENT 제외. CARRYOVER는 충족 1회 인정.
```

테스트:
- TC1 본인조 ATTEND → 충족+1
- TC2 사전이월 → 6/12 버튼 [CARRYOVER,OTHER_GROUP], 충족+1
- TC3 사후이월 → 과거 미참여일 PAST_ABSENT 후보로 뜸
- TC4 이미 참여한 과거일 → 후보 제외
- TC5 타조참 → 충족 변화 없음
- TC6 이월취소(기록삭제) → target [ATTEND,ABSENT] 복구
- TC7 당월 경계 → 다음달 활동 후보 제외
- TC8 자격 다 씀 → 후보 빈 배열
